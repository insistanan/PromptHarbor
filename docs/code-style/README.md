# 代码规范索引

本目录存放项目的详细代码规范。根目录的 `AGENTS.md` 和 `CLAUDE.md` 只保留协作入口、项目约束和规范索引，不承载过长的实现细节。

## 文件说明

- `rust.md`：Rust 后端、hook 采集器、SQLite、错误处理和日志规范。
- `frontend.md`：Tauri 前端、React、Milkdown、状态管理和界面交互规范。
- `docs.md`：Markdown 文档、设计方案、ADR 和注释写作规范。

## 通用原则

- 代码要围绕 PromptBox 的本地优先、低打扰、可恢复采集来设计。
- 优先使用项目已有模式，不为短期便利引入不必要抽象。
- 数据边界要清晰：默认只记录用户 prompt，不记录模型回复，不自动上传。
- 失败路径要可解释、可恢复，尤其是 hook 采集链路。
- 文件和模块命名应直接表达职责，避免宽泛名称。

## 质量要求

- 用户明确要求实现代码时，改动应尽量小而完整。
- 用户明确要求测试或验证时，优先运行与改动范围直接相关的检查。
- 没有用户明确要求时，不主动运行代码、编译或测试。
- 对涉及 hooks、数据库 schema、数据迁移和删除能力的改动，要额外说明风险和回滚方式。

## 同步要求

若本目录中的规范变更影响 agent 协作入口，必须同步更新根目录的 `AGENTS.md` 与 `CLAUDE.md`。这两个文件必须保持内容一致。
