# MVP 优先验证本地提示词工作流

PromptBox 的 MVP 必须覆盖写草稿、复制草稿、用户在 Agent 客户端真实提交、hook 捕获、内容一致后清空草稿、历史回看这一完整最小闭环。自动总结、提示词优化和“坏了我成替身了”人设提炼等大模型增强能力延后实现，因为它们会过早引入外部 API、隐私授权、成本控制和更复杂的产品边界。

**Considered Options**

- 采集优先 MVP：先做 hook 和历史记录，编辑体验后补；风险是无法验证 PromptBox 的核心编辑工作流。
- 编辑优先 MVP：先做 Milkdown 草稿窗口，采集后补；风险是无法验证真实提交和历史记录闭环。
- 完整最小闭环 MVP：先验证本地提示词工作流，再扩展增强能力。

**Consequences**

- MVP 要同时包含 hook 采集、会话登记、Milkdown 草稿、复制草稿、hook 确认清空和历史回看。
- MVP 不包含自动总结、提示词优化、人设提炼、历史导入、自动注入终端和独立弹出工作区窗口。
- 后续大模型 API 模块必须建立在本地数据边界和用户显式授权之上。
